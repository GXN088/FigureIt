Классы Triangle, Rectangle и Circle — геометрические фигуры, поэтому они и унаследованы от класса Shape.
Переопредели в них метод printInfo(), чтобы в консоли выводилось название конкретной фигуры:
Для Triangle — "Треугольник";
Rectangle — "Прямоугольник";
Circle — "Круг".

Метод main() в тестировании участия не принимает.

Псевдокод:

```
// Базовый класс Фигура
КЛАСС Фигура:
    МЕТОД вывестиИнформацию():
        ВЫВЕСТИ "Геометрическая фигура"
    КОНЕЦ МЕТОДА
КОНЕЦ КЛАССА

// Класс Круг наследуется от Фигуры
КЛАСС Круг НАСЛЕДУЕТ Фигура:
    МЕТОД вывестиИнформацию():
        ВЫВЕСТИ "Круг"
    КОНЕЦ МЕТОДА
КОНЕЦ КЛАССА

// Класс Прямоугольник наследуется от Фигуры  
КЛАСС Прямоугольник НАСЛЕДУЕТ Фигура:
    МЕТОД вывестиИнформацию():
        ВЫВЕСТИ "Прямоугольник"
    КОНЕЦ МЕТОДА
КОНЕЦ КЛАССА

// Класс Треугольник наследуется от Фигуры
КЛАСС Треугольник НАСЛЕДУЕТ Фигура:
    МЕТОД вывестиИнформацию():
        ВЫВЕСТИ "Треугольник"
    КОНЕЦ МЕТОДА
КОНЕЦ КЛАССА

// Главная программа
НАЧАЛО ПРОГРАММЫ:
    СОЗДАТЬ объектФигура = НОВАЯ Фигура()
    СОЗДАТЬ объектКруг = НОВЫЙ Круг()
    СОЗДАТЬ объектПрямоугольник = НОВЫЙ Прямоугольник()
    СОЗДАТЬ объектТреугольник = НОВЫЙ Треугольник()
    
    объектФигура.вывестиИнформацию()
    объектКруг.вывестиИнформацию()
    объектПрямоугольник.вывестиИнформацию()
    объектТреугольник.вывестиИнформацию()
КОНЕЦ ПРОГРАММЫ
```

Объяснение:

· Это пример полиморфизма в ООП
· Базовый класс Shape (Фигура) определяет общий метод printInfo()
· Классы-наследники (Circle, Rectangle, Triangle) переопределяют этот метод
· Каждый класс выводит свое специфическое название фигуры
· При вызове метода у каждого объекта будет выполнена соответствующая версия метода

Ожидаемый вывод:

```
Геометрическая фигура
Круг  
Прямоугольник
Треугольник
```
